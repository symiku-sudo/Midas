# Refactor（重构）

## 1) 必须流程
1. 定义边界：明确“会改什么/不会改什么”。
2. 给出分阶段方案，保证每一步都可回滚。
3. 实施结构优化，保持输入输出与接口契约一致。
4. 补/改测试证明行为不变。
5. 运行验证并记录关键结果。
6. 输出交付摘要与风险说明。

## 2) 结束门禁（任一不满足都不能结束）
- 未明确行为不变边界，不能结束。
- 无前后行为一致性证据（测试/接口验证），不能结束。
- 引入了无关功能或业务逻辑变更，不能结束。
- 影响接口层但未回归关键路由，不能结束。

## 3) Midas 硬约束
- 不改变 API 外部行为和错误码语义。
- 不破坏统一响应协议：`ok/code/message/data/request_id`。
- 若重构涉及接口层，必须回归验证 `health`、`bilibili`、`xiaohongshu` 路由。
- 若重构涉及数据层或清理脚本，默认不得删除/覆盖 `server/.tmp/backups/` 下 `.db` 备份（含 `*_latest.db`）。

## 4) 统一交付模板
- 重构边界：明确改动内外范围。
- 关键改动：列出主要结构调整与受影响模块。
- 一致性证据：列出测试/接口验证结果。
- 风险与回滚：说明潜在回归点与回滚路径。
- 文档同步：如涉及目录/职责变化，说明更新情况。
