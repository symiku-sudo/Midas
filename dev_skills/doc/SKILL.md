# Doc（文档维护）

## 1) 必须流程
1. 明确目标读者、文档范围和 DoD。
2. 先读取相关文档，避免冲突、重复或术语漂移。
3. 仅做需求相关改动，保持现有结构和命名风格。
4. 校对命令、路径、错误码、接口名与配置项一致性。
5. 对示例命令做可执行性验证，或标注前置条件。
6. 输出变更摘要与验证说明。

## 2) 结束门禁（任一不满足都不能结束）
- 术语与代码/契约不一致，不能结束。
- 行为变化已落地但文档未同步，不能结束。
- 示例命令不可执行且未标注前置条件，不能结束。
- 复盘文档缺关键字段（时间线/证据/防再发），不能结束。

## 3) Midas 文档优先级
1. `server/API_CONTRACT.md`
2. `server/README.md`
3. `README.md`
4. `dev_skills/doc/incidents/*.md`

## 4) 事故复盘文档规范
- 文件路径：`dev_skills/doc/incidents/YYYY-MM-DD-<topic>.md`。
- 最小结构：
1. 背景与影响。
2. 时间线（必须使用绝对日期时间）。
3. 根因链路。
4. 证据。
5. 处置与回滚。
6. 防再发动作（Owner + 截止日期）。

## 5) 统一交付模板
- 变更文件列表。
- 关键改动摘要。
- 验证说明（执行了哪些示例命令/核对项）。
- 未完成项与后续建议（如有）。
