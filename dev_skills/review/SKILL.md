# Review（代码审查）

## 审查维度
- 逻辑正确性与边界条件
- 错误处理与错误码一致性
- 安全与敏感信息处理
- 配置与可运维性
- 备份保护：是否误删/覆盖 `server/.tmp/backups/` 下 `.db` 备份（含 `*_latest.db`）
- 测试覆盖和回归风险
- 文档一致性（README/API_CONTRACT/WEEKLY_PROGRESS）

## 输出格式
1. 高优先级问题（必须修）
2. 中优先级问题（建议修）
3. 低优先级优化（可选）
4. 建议新增测试

## 说明
- 尽量提供文件路径与行号定位。
- 若无问题，明确说明“未发现阻断问题”，并给出残余风险。
